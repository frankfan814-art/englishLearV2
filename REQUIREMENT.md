# 英语单词朗读学习 APP - 需求文档

## 1. 产品定位

一款极简的英语单词朗读学习工具，核心功能是**自动朗读单词、显示释义、一轮轮刷词**。不做登录、不做社交、不做复杂算法，专注"听读刷词"这一件事。

---

## 2. 目标用户

- 有一定英语基础，想通过大量听读快速过词的人
- 不想被复杂功能干扰，只想"开着就听、看着就记"的人
- 适合通勤、做家务等场景下被动听读

---

## 3. 核心功能

### 3.1 单词卡片展示

每张卡片包含以下信息，同时展示（不做逐步揭示，直接全部显示）：

| 元素 | 说明 |
|------|------|
| 单词 | 大字体，居中显示，如 `abandon` |
| 音标 | 美式音标，如 `/əˈbændən/` |
| 中文释义 | 词性+释义，如 `v. 放弃；抛弃` |
| 例句 | 1个常用例句+中文翻译 |

### 3.2 自动朗读

- **默认开启**自动朗读模式
- 每个单词停留 **1秒**，自动切换到下一个
- 朗读内容：读单词发音（不读释义和例句）
- 用户可随时暂停/继续

### 3.3 手动切换

- 支持手动点击"上一个"/"下一个"切换单词
- 手动切换后，自动朗读继续从当前单词往下走
- 支持滑动切换（移动端手势）

### 3.4 轮次进度

- **一轮 = 全部单词列表（约2万个）从头到尾过一遍**
- 显示当前轮次中已读单词数 / 总数，如 `第1轮 352 / 20000`
- 一轮读完后自动进入下一轮，轮次计数+1
- 记录历史完成轮次数

### 3.5 进度持久化

- 本地存储当前轮次进度（当前读到第几个单词）
- 本地存储已完成轮次数
- 关闭 APP 后重新打开，从上次位置继续
- **不需要登录，不需要云端同步**

## 4. 单词数据

### 4.1 词库来源与筛选策略

**目标：约2万个常用但非过于简单的英语单词**

筛选逻辑：
1. 以 **ECDICT**（77万词主库）为基础，取 BNC/COCA 频率排名前 20000 的词
2. 过滤掉 **中考词汇**（约1500个最基础词，如 the, of, open, go 等）
3. 过滤掉 **功能词/停用词**（介词、连词、代词等约127个，如 of, the, and, in）
4. 过滤掉 **Collins 5星词**（约680个最高频词，几乎全是功能词和极简单词）
5. 不足2万的部分，从考试词汇（CET-4/6、高考、IELTS、TOEFL）中补充

**预期结果：约 15000-20000 个有实际学习价值的单词**

### 4.2 单词数据字段

每个单词需包含：

```
{
  "word": "abandon",
  "phonetic": "/əˈbændən/",
  "pos": "v.",
  "definition": "放弃；抛弃",
  "example": "He abandoned his car in the snow.",
  "example_translation": "他把车丢弃在雪地里。",
  "tag": "cet4"  // 来源标签，可选值：cet4, cet6, gk, ielts, toefl, gre
}
```

### 4.3 数据来源（开源项目）

| 数据 | 推荐来源 | 说明 |
|------|----------|------|
| 主词库 | [ECDICT](https://github.com/skywind3000/ECDICT) (MIT) | 77万词，含音标、释义、频率、考试标签 |
| 例句 | [kajweb/dict](https://github.com/kajweb/dict) | 含例句+翻译，按考试分类 |
| 音频文件 | [ismartcoding/endict](https://github.com/ismartcoding/endict) | 约8000个MP3发音文件 |
| 发音API | 有道词典API（备用） | `https://dict.youdao.com/dictvoice?audio={word}&type={1|2}` |

### 4.4 发音方案

**优先方案：Web Speech API（浏览器原生TTS）**
- 零成本，无需API，离线可用
- 质量：现代浏览器效果可接受
- 实现：`window.speechSynthesis`

**备用方案：有道词典发音API**
- URL格式：`https://dict.youdao.com/dictvoice?audio=abandon&type=2`
- type=1 美式，type=2 英式
- 需要网络，音质更好

**最终方案：优先使用本地预下载音频（endict的MP3），缺失的词用Web Speech API兜底**

## 5. 界面设计

### 5.1 整体布局（单页面，极简）

```
┌─────────────────────────────┐
│  第1轮  352 / 20000    ⚙️   │  ← 顶部：轮次进度 + 设置入口
│                             │
│         abandon             │  ← 单词（大字体）
│        /əˈbændən/          │  ← 音标
│                             │
│       v. 放弃；抛弃          │  ← 释义
│                             │
│  He abandoned his car       │  ← 例句
│  in the snow.               │
│  他把车丢弃在雪地里。        │  ← 例句翻译
│                             │
│   ◀️    ▶️/⏸️    ▶️▶️       │  ← 控制栏：上一个 | 播放/暂停 | 下一个
│                             │
│  ━━━━━━━━━━━░░░░░░░░░░░░   │  ← 底部进度条
└─────────────────────────────┘
```

### 5.2 设置页面（极简）

- 朗读速度：0.5秒 / 1秒 / 1.5秒 / 2秒 / 3秒（默认1秒）
- 发音类型：美式 / 英式（默认美式）
- 重置进度：清除当前轮次进度，从第1个词重新开始

### 5.3 设计原则

- **单页应用**，无路由跳转
- 深色/浅色主题跟随系统
- 字体大小适配移动端
- 无广告、无弹窗、无引导页

## 6. 技术方案

### 6.1 技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 框架 | React + Vite | 轻量快速，生态成熟 |
| 样式 | Tailwind CSS | 快速开发，移动端适配方便 |
| 存储 | localStorage | 极简，无需后端 |
| 发音 | Web Speech API + 本地音频 | 离线优先 |
| 部署 | 静态部署（Vercel/Netlify） | 免费，无需服务器 |
| 打包 | PWA（可选） | 可安装到手机桌面 |

### 6.2 数据加载策略

- 单词数据按 **分片加载**（如每1000个词一个JSON文件），避免首屏加载2万条数据
- 当前片+前后各1片预加载，保证切换流畅
- 例句数据单独分片，按需加载

### 6.3 本地存储结构

```json
{
  "currentRound": 1,
  "currentIndex": 352,
  "completedRounds": 0,
  "settings": {
    "speed": 1,
    "accent": "us"
  }
}
```

## 7. 不做的功能（明确排除）

| 功能 | 原因 |
|------|------|
| 用户登录/注册 | 不需要，本地存储即可 |
| 云端同步 | 增加复杂度，非核心需求 |
| 艾宾浩斯遗忘曲线 | 过于复杂，与"刷词"定位不符 |
| 单词收藏/标记 | 可以后续加，当前不做 |
| 测试/考试模式 | 非核心需求 |
| 社交/排行榜 | 非核心需求 |
| 词根词缀分析 | 非核心需求 |
| 多词书选择 | 只有一本2万词书 |
