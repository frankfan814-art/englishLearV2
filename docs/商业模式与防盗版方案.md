# 「单词朗读」商业化方案：定价、销售与防分享

> 版本：v1 草稿（供讨论）
> 目标：把现有 App 以 **¥18.8 买断制** 直接卖给终端用户，并尽量防止"买一份、到处转发"。

---

## 1. 产品现状盘点

| 维度 | 现状 | 对商业化的影响 |
|------|------|----------------|
| 形态 | Capacitor 打包的 Android APK，纯前端 SPA | 可直接发 APK 卖，无需上架商店 |
| 后端 | **无**（无账号、无服务器、无支付） | 防分享方案要么纯本地，要么加轻量后端 |
| 数据 | 词库 JSON 分片明文打包在 APK 内（`public/data/`） | 任何人解压 APK 即可拿走全部词库，需要考虑加密 |
| 支付 | 无 | 卖 APK 走站外收款（微信/闲鱼），或上商店走官方支付 |
| 合规 | 无软著、无 App 备案 | 上国内商店需要资质；私下分发卖不需要，但规模大了要注意 |

**核心卖点（写销售文案时用）：**
- 自动朗读刷词：单词 → 释义 → 例句全自动播放，开车/通勤/睡前"磨耳朵"
- 4 国语言：英 / 日 / 韩 / 德，46,000+ 词
- 分级词表：托福 / GRE / 四六级 / 考研 / 雅思 / 高考 / JLPT N5–N1
- 离线词库 + 已掌握跳过 + 多轮复习进度
- 无广告、无登录、不收集任何数据

## 2. 定价分析：¥18.8 买断

### 2.1 竞品价格锚点（大致区间，以官方实时价格为准）

| 产品 | 模式 | 价格 |
|------|------|------|
| 百词斩 | 免费 + 广告/周边 | 0 |
| 墨墨背单词 | 免费 + 词量上限付费解锁 | 约 ¥12 起，多档，深度用户累计上百元 |
| 不背单词 | 免费 + 会员订阅 | 会员约 ¥30/月起 |
| Anki | 开源免费（Android）/ AnkiMobile 付费（iOS）| iOS 约 $24.99 |
| List 背单词等小工具 | 买断 | ¥6 ~ ¥25 |

### 2.2 结论

- **¥18.8 处于"一杯奶茶"的冲动消费区间**，买断制对工具类小 App 是合理的，用户决策成本低。
- 你的差异化不是"比百词斩强"，而是**"自动朗读刷词"这一个场景打透**——竞品里做这个形态且做得极简的很少。文案上不要拼词库量，拼"解放眼睛、磨耳朵"。
- 建议价格策略：
  - **标准价 ¥18.8**（单设备激活）
  - 首发/促销可放 **¥9.9 早鸟价** 冲第一批口碑和买家秀
  - 后续上云激活后可加 **¥28.8 双设备档**（见 4.3）
- 不要定太低（<¥9.9）：低价不会显著多卖，反而拉低"这是正经软件"的心智，且售后成本不变。

## 3. 销售渠道

按"今天就能开卖"到"长期正规军"排序：

### 3.1 微信私域直卖（第 1 天就能做，推荐起步）
- 朋友圈 / 微信群 / 公众号引流 → 加微信 → 收款码收 ¥18.8 → 发 APK + 激活码（配合 4.2 方案）
- 优点：零成本、零资质、即时到账；缺点：纯手动，量大了累

### 3.2 闲鱼 / 淘宝虚拟商品
- 挂"背单词软件 安卓版"商品，买家拍下后**自动发货 APK 下载地址 + 激活码**（闲鱼有自动发货工具，淘宝虚拟商品可用第三方发货插件）
- 注意：闲鱼对"虚拟商品/软件"类目有一定限制，标题不要写"破解/永久"等敏感词；淘宝虚拟类目需缴纳保证金（约 ¥1000）
- 蒲公英（你 CI 已经接了）可作为 APK 下载页，链接直接发给买家

### 3.3 内容平台引流（放大销量的关键）
- 小红书 / 抖音 / B站：录 15 秒"开车时自动朗读刷词"的使用场景视频，评论区引导私信
- 这类"场景化演示"比参数罗列转化高得多，是 18.8 价位产品的主要放量方式

### 3.4 Google Play（海外，可选）
- 开发者账号一次性 $25，可定价付费下载或走 Play 内购
- 你的 TTS 链路（有道/百度公开接口）海外可用性未验证，且 Play 审核对"使用非官方接口"无感，但需隐私政策页
- 建议国内卖顺了再考虑

### 3.5 国内应用商店（华为/小米/OPPO/vivo，长期）
- 需要：**软件著作权** + **App 备案（工信部）** + 企业或个体工商户开发者资质（部分商店个人开发者类目受限）
- 周期长（1–3 个月），适合销量验证之后再走

## 4. 防分享 / 防盗版方案（重点）

### 4.1 先认清现实（重要）

1. **APK 可以被任意转发**，转发后对方直接安装就能用——这是要防的主要场景。
2. **任何纯客户端校验都能被破解**：APK 用 jadx/apktool 反编译后，懂行的人可以删掉激活校验逻辑重打包，或抠出密钥。墨墨背单词有完整账号体系照样破解版满天飞。
3. 所以目标要现实：**防"随手转发给同事/群发给别人"，不防职业破解者。** 对 ¥18.8 的产品，只要让"白嫖"变得比"花 18.8"更麻烦，绝大多数真实用户就会付钱。
4. 防护强度每加一级，开发和维护成本也加一级。按下面的路线逐级投入，**不要第一天就上最重方案**。

### 4.2 方案 A：离线一机一码（推荐起步，零成本，1–2 天开发）

**原理：激活码 = 对设备指纹的签名，换一台手机码就失效。**

```
买家手机                    你（卖家）
   │  首次打开 App，显示本机"设备码"（如 3K7D-9Q2X）
   ├──────────────────────────────►│
   │                               │ 运行 scripts/gen_license.js 3K7D9Q2X
   │                               │ 算出激活码 = HMAC_SHA256(密钥, 设备码) 取前 16 位
   │  微信发回激活码（如 V8F2-K4MN-P7QW-A3Z9）
   │◄──────────────────────────────┤
   │  App 内用同样的密钥和算法校验，一致 → 写 localStorage 标记已激活
```

**实现要点（结合现有代码）：**
- 设备指纹：用 `@capacitor/device` 的 `getId()`（Android 上即 ANDROID_ID，重装 App 不变，恢复出厂设置才变）。不要自己生成 UUID 存 localStorage——用户清数据/重装就会变，售后会把你烦死。
- 密钥：一个随机长字符串，编译进前端代码（分散拼接存放 + 混淆，见 4.5）。卖家侧只在 `scripts/gen_license.js` 里放一份，**绝不外传这个脚本和密钥**。
- 激活态存储：localStorage 加一个键（如 `vocab_license`），`src/store/useAppStore.ts` 的 `initialize()` 最前面校验，未激活走"试用模式"（见 4.4）或激活页。
- 校验入口：新建 `src/utils/license.ts`：读取设备码 → 本地算 HMAC → 与用户输入的激活码比对。Web Crypto API 自带 HMAC-SHA256，无需引入依赖。
- 卖家工具：`scripts/gen_license.js`，命令行 `node scripts/gen_license.js <设备码>` 输出激活码，10 行代码。

**能防什么 / 不能防什么：**

| 场景 | 结果 |
|------|------|
| 买家把 APK 转发给朋友 | ✅ 朋友设备码不同，激活码无效，等于白装 |
| 买家连激活码一起转发 | ✅ 同上，码在别的机器上算不出来 |
| 反编译删校验、抠密钥重打包 | ❌ 防不住（靠 4.5 提高门槛，但无法杜绝） |
| 买家恢复出厂设置 | ⚠️ 设备码会变，需要售后补码（写进售后政策） |

**成本：** 0 元，无服务器，无网络依赖。

### 4.3 方案 B：激活码 + 轻量云校验（销量起来后升级）

方案 A 的两个痛点——**每单要手动算码**、**换机要人工补码**——在单量上来后会很烦。升级方式：

1. 预生成一批随机激活码（如 `VCAB-XXXX-XXXX-XXXX`）存进云数据库，卖一个发一个（闲鱼/淘宝自动发货直接发卡密）。
2. App 首次激活时联网调云函数：校验激活码存在 → 绑定当前设备 → 记录绑定数（上限 2 台，即"单设备价 18.8，双设备 28.8"也能这样实现）。
3. 换机：用户在旧机点"解绑"或你后台手动解绑，新机再用同一激活码激活。

**技术选型（都是低代码低成本的）：**

| 方案 | 成本 | 说明 |
|------|------|------|
| 微信云开发（云函数+云数据库） | 有免费额度，超出按量（个人小量 ≈ 0~20 元/月） | 国内访问快，不需要域名备案，推荐 |
| LeanCloud | 有免费开发版 | 接入简单，免费版有请求限制 |
| 阿里云/腾讯云 函数计算 + 表格存储 | 按量付费，小量几乎为 0 | 需要一点云账号配置 |

**新增好处：** 激活码可随时作废（发现被倒卖就封）、能看真实销量、支持自助换机。
**代价：** 首次激活需要联网；要维护一点点后端代码；云账号要实名。

### 4.4 配套设计：试用模式（强烈建议）

不要做成"不激活就一块白板"，而是：

> **未激活可试用：开放每种语言前 200 词 + 全部功能；激活后解锁完整词库。**

理由：
- 买家付款前能真实体验"自动朗读"的核心爽点，转化率显著高于盲买。
- 闲鱼/淘宝买家常要求"先试试"，试用版天然满足，减少售前沟通。
- 即使将来出现破解，破解者传播的也是"功能完整的收费版"——试用机制不影响这个现实，但能把摇摆用户先转化为付费。

实现很轻：`src/store/useAppStore.ts` 里 `wordIndexesInList` 在未激活时截断为前 200 个索引即可，激活页用现有 Drawer 组件做。

### 4.5 配套加固（无论 A/B 都建议做）

按性价比排序：

1. **词库加密（中性价比，防同行抠词库二次售卖）**
   现状：词库是明文 JSON，解压 APK 就能拿走 4.6 万词。可用 AES 加密分片（构建时加一个加密脚本），`src/utils/dataLoader.ts` 加载后内存解密，密钥编译进代码。会让 `DataLoader` 多一步解密，性能影响可忽略。
2. **构建加固**：`vite.config.ts` 确认关闭 sourcemap（`build.sourcemap: false`）、terser 开启 `drop_console`；已默认 minify。
3. **JS 混淆（可选）**：`javascript-obfuscator` 对 license 校验相关代码混淆。注意会让包变大、可能引入诡异的运行时 bug，**只在激活功能稳定后加**，且要真机回归测试。
4. **校验逻辑分散**：不要把激活判断集中在一个 `if` 里（破解者改一处就全解锁）。在"超过 200 词加载""词表切换"等多处自然地带出校验。这属于"恶心人"层面，不能根治，但显著提高重打包时间。

### 4.6 方案对比与推荐路线

| | 方案 A 离线一机一码 | 方案 B 云激活 | 方案 C 商店官方支付 |
|---|---|---|---|
| 开发量 | 1–2 天 | 3–5 天 | 1–2 周（资质更久） |
| 金钱成本 | 0 | ≈0–20 元/月 | $25（Play）/ 软著+备案 |
| 防随手转发 | ✅ | ✅ | ✅ |
| 防破解重打包 | ❌ | ❌（稍好） | ❌ |
| 每单人工操作 | 要手动算码 | 自动发货 | 全自动 |
| 换机售后 | 人工补码 | 自助解绑 | 商店账号体系 |
| 需要联网 | 不需要 | 首次激活需要 | 需要 |

**推荐路线：**

```
第一阶段（现在就做）：方案 A + 试用模式 + 词库加密 + 构建加固 → 微信/闲鱼开卖
第二阶段（月销 > 50 单，或换机补码每周超过几单）：升级方案 B，激活码自动发货
第三阶段（想上规模/品牌）：软著 + 备案 + 国内商店 / Google Play
```

## 5. 收款与合规

- **个人收款码小额经营**：微信/支付宝个人收款码收这种小额偶发收入，实务中普遍可行；但持续规模化经营（月入数万）建议注册**个体工商户**（线上可办，几乎零成本），并依法申报。
- **软件著作权**：私下卖 APK 不需要；上国内商店需要。自己办约 ¥300 官费、1–2 个月，代办 ¥800–1500。
- **App 备案**：上国内商店/用国内云服务分发正式版时需要（工信部 2023 年起要求）。私下发 APK 安装不涉及。
- **词库版权确认**：`public/data/` 词库来自开源词典数据（见 `ecdict_research_report.md`），商用前请逐份确认源数据 License 允许商业使用；日语 JLPT 标注、韩德语词源同理。这是**真正的法律风险点**，比防盗版更值得先确认。
- **TTS 接口风险**：英语发音走有道公开音频接口、部分走百度公开 TTS 接口，均为**非官方授权**的公开端点。商用 App 依赖它们有两层风险：(1) 对方随时可能限流/封禁；(2) 严格讲违反其服务条款。建议在销售文案里不要宣传"有道发音"，并在设置里保留系统 TTS 兜底；中期可考虑接入正规 TTS 服务（讯飞/阿里云，按量付费，单词朗读场景成本极低）。

## 6. 售后政策（写在自动回复/商品详情里）

- **换机**：方案 A 下凭购买记录免费补发新机激活码，每单限补 3 次（防无限转卖）；方案 B 后改为自助解绑。
- **恢复出厂设置**：设备码会变，等同换机处理。
- **退款**：虚拟商品售出不退，但提供试用版先体验（用试用模式挡住"买了不满意"的纠纷）。
- **服务**：建一个买家微信群，TTS 接口失效、词库修正等问题统一在群里公告 + 发新版 APK。

## 7. 版本更新策略

- 买断制 = 终身更新会拖垮你，建议写明：**¥18.8 含 1 年更新**，之后老版本可永久使用，新版（如新增语言/词库）老用户优惠价。
- 发版渠道沿用现有 CI：push main → 自动构建 APK → 蒲公英，买家群/商品页给固定下载链接。
- 激活码版本兼容：激活态存 localStorage，覆盖安装不丢（同一设备码算出来一致），用户无感升级。

## 附：关于"用户买了到处发"的心态

最后说句实在话：¥18.8 的工具软件，**盗版的损失往往是被高估的**——会转发破解版的人大概率本来就不会付这 18.8。方案 A 的意义不是堵住所有漏洞，而是：

1. 让转发这条路径"断了"（码在别的机器上没用），把摇摆用户推回付费；
2. 让买家感觉到"我买的是一份正经软件"，值这个价；
3. 给你留下真实的销量数据（发了多少码 = 卖了多少份），这是后续所有运营决策的基础。

把省下来的精力花在引流和内容上，比堆防护更赚钱。
